# Style recommendations — clean & professional, stand out

## What we changed (this pass)

### Work page
- **Tech pills:** Centered in the card; style updated to solid pills (neutral background, pill radius, no busy border) so they read as tags, not form chips.
- **Card content:** Title, description, tech stack, and CTA are centered so the page feels balanced instead of everything left-aligned.
- **CTA button:** Centered, not full-width; gradient (violet → cyan) kept so it stays a clear primary action.

### About page
- **Focus area cards:** Replaced the **thick left border** with a **thin top accent** (2px brand) and even padding. Cards read as proper content cards instead of “quote” blocks, which reads more professional and less template-y.
- **Section titles:** “Focus Areas”, “Technical Skills”, intro title, etc. are centered so the layout feels intentional.
- **Skills & Technologies:** Tags are centered instead of left-aligned to match the rest of the page and reduce left-heavy alignment.

---

## Opinion: side-color containers

**Left-border accent cards** are a known pattern (Trello, Notion, etc.) but can feel:
- Like a “quote” or sidebar block
- A bit default when overused

**Alternatives we used:**
- **Focus areas:** Thin **top** accent (2px) + soft gradient background. Same brand hint, cleaner card.
- **Technical cards:** Already neutral cards; we kept them with a subtle cyan tint and no bar so they stay minimal.

If you want to try other options later:
- **No accent:** Just 1px border + shadow (very minimal).
- **Corner accent:** Small colored corner (e.g. 4px × 4px) instead of a full edge.
- **Icon only:** Small icon next to the title instead of a bar.

---

## What keeps the site clean and professional

1. **Consistent alignment** — Centered headings and key blocks; body text and lists can stay left for readability.
2. **One clear accent** — Violet + cyan used for CTAs and light tints; not every element is “accented.”
3. **Solid components** — Tech pills and tags are solid and simple; gradients reserved for primary actions and section backgrounds.
4. **Cards as cards** — Focus/technical areas use full borders (or top accent) and padding, not heavy side bars, so they read as content cards.
5. **Whitespace** — Section spacing and grid gaps unchanged so the new alignment doesn’t crowd the layout.

---

## Optional next tweaks (if you want to go further)

- **Blog / Gallery:** Add the same centered section titles and card treatment if they have similar blocks.
- **Nav active state:** Gradient underline (violet → cyan) on active link for a bit more polish.
- **Footer:** One subtle gradient or tint in the footer strip to tie it to the rest of the palette without competing.
