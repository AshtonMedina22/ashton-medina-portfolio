## Problem

Customer operations, account management, permitting cycles, vendor follow-ups, compliance reviews, and executive inbox work all generate repetitive but high-risk correspondence. Teams want to use LLMs and AI assistants to draft this communication, but directly wiring AI or automated email sends into operational records creates unacceptable risk:

- Credentials and keys could leak to the browser or client-side scripts.
- AI-generated drafts could miss critical source context or reference the wrong records.
- Fully automated sends could bypass the human judgment required for customer, partner, internal, and leadership communications.

The organization needed a secure AI-assisted communication pipeline with server-side controls, human-in-the-loop review, and operational governance, not fire-and-forget automation.
  </div>
  <div>
## Solution

I designed an event-driven AI-assisted review gateway that turns database events, spreadsheet changes, and operational triggers into structured, reviewable email drafts behind a secure server boundary.

- Apps Script and Edge/Node routes normalize source data into a strict draft contract that can be passed into an LLM-powered drafting step or rule-based generator.
- AI/automation work and email draft creation run server-side behind OAuth-controlled service accounts and mailbox boundaries.
- Every draft flows into a human-in-the-loop review desk where operators compare AI-assisted copy against the source record and decide whether to send, revise, or discard.

The result is a secure AI-assisted communication pipeline with AgentOps-style guardrails: event-driven automation, server-side security, and mandatory human approval.

## How It Works

<HowItWorksGrid>
<HowItWorksCard title="Installable Spreadsheet Triggers">
Apps Script watches specific row and status changes, then sends normalized metadata to the platform backend instead of embedding privileged workflow or AI logic in the sheet. This keeps AI-assisted drafting and decision logic out of end-user spreadsheets and inside a controlled backend.
</HowItWorksCard>

<HowItWorksCard title="Database Event Triggers">
Supabase and PostgreSQL-style event hooks stream operational records into the same AI-assisted review pipeline, giving spreadsheets and databases a shared automation and AI contract. Both sources feed a consistent event-to-normalize-to-draft-to-review lifecycle that can extend into broader MLOps and AgentOps orchestration.
</HowItWorksCard>

<HowItWorksCard title="Server-Side Access Isolation">
The browser never receives Google private keys, OAuth refresh tokens, Workspace impersonation settings, or privileged service credentials. All sensitive work, including LLM calls, AI-assisted drafting, and email draft creation, happens in Edge or Node server routes while enforcing mailbox and tenant boundaries.
</HowItWorksCard>

<HowItWorksCard title="Deterministic Draft Contract">
Raw operational fields are converted into a typed draft payload. Customer names, account IDs, dates, owners, request types, and due dates are passed as explicit fields, not free-form text. This contract becomes the input schema for AI or rule-based generators, improving prompt reliability, traceability, and safety.
</HowItWorksCard>

<HowItWorksCard title="Email Draft Worker">
The backend creates drafts through Google Workspace APIs only after OAuth consent or Workspace delegation is available. Draft IDs and metadata are written back to the source record for traceability into who triggered what, when, and with which context.
</HowItWorksCard>

<HowItWorksCard title="Human-in-the-Loop Review Desk">
System output is always treated as a draft, never an automatic send. Operators review the AI-assisted copy against the source fields, then choose to send, revise, or discard, creating a human-in-the-loop control layer for AI-generated communications.
</HowItWorksCard>
</HowItWorksGrid>
