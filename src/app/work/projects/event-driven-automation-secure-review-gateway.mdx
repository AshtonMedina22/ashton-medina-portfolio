## Problem

Customer operations, account management, permitting or approval cycles, vendor follow-ups, compliance reviews, and executive inbox work create repetitive correspondence that still requires precision. Directly wiring automated email sends into operational records would expose too much risk: private keys could leak to the browser, draft details could miss source context, and automated sends could bypass the human judgment needed for customer, partner, internal, and leadership communications.
  </div>
  <div>
## Solution

An event-driven scripting trigger and secure server-side review gateway converts database events, spreadsheet changes, and operational triggers into structured, reviewable drafts. Apps Script and Edge/Node routes pass normalized fields into a constrained draft contract, while email draft creation stays behind OAuth-controlled server code and requires a human review step before anything leaves the system.

## How It Works

<HowItWorksGrid>
<HowItWorksCard title="Installable Spreadsheet Triggers">
Apps Script watches specific row and status changes, then sends structured metadata to the platform backend instead of embedding privileged workflow logic in the sheet.
</HowItWorksCard>

<HowItWorksCard title="Database Event Triggers">
Supabase and PostgreSQL-style event hooks can move operational records into the same review pipeline, giving spreadsheet and database sources a shared automation contract.
</HowItWorksCard>

<HowItWorksCard title="Server-Side Access Isolation">
The browser never receives Google private keys, OAuth refresh tokens, Workspace impersonation settings, or privileged service credentials. All sensitive work happens in Edge or Node server routes.
</HowItWorksCard>

<HowItWorksCard title="Deterministic Draft Rules">
Raw operational fields are converted into a fixed draft payload. Customer names, account IDs, dates, owners, request types, and due dates are passed as explicit values instead of free-form context.
</HowItWorksCard>

<HowItWorksCard title="Email Draft Worker">
The backend creates drafts through Google Workspace APIs only after OAuth consent or Workspace delegation is available. Draft IDs can be written back to the source record for traceability.
</HowItWorksCard>

<HowItWorksCard title="Human Approval Desk">
System output is treated as a draft, not an action. Operators review the prepared copy, compare it against source fields, and decide whether to send, revise, or discard it.
</HowItWorksCard>
</HowItWorksGrid>
